Definitions

Tracks

Tracks represent categories of events that will be grouped together on the booking site. Each event type is associated with a single track (although event types can have the same names - e.g. an "aerial class" event type on the Adults track is different to an "aerial class" event type on the Kids track.

Events in each track will be shown on a different page. Tracks with no scheduled events will not be visible on the site.

For example, you might have a track for Adults classes, Youth classes, Workshops, Parties etc. As tracks will be displayed as menu options, it's best to try to choose concise descriptive names for them.

Choose one "default" track, which will be the landing page for the site.

Event Types

Event Types define general characteristics about an event, such as the number of participants and cancellation period. Scheduled events for particular days and times are generated from Event Types and will share those characteristics. An Event Type must be associated with one Track.

Event Types are used to define pricing for events, and can be associated with multiple Credit Block. For example, you might create an Aerial Class event type, in the Adult Classes track. You can then set up multiple Credit Blocks attached to this event type to allow students to purchase credit in blocks of 1, 4 or 8 classes.

Events

An Event is a single, scheduled event, which has a specific date and time, and which students can book into.

Course Types

Course Types define the outline of a Course, and hold the Event Type and the number of events in the course. They can be associated with Credit Blocks for courses, which will only allow students to book for classes in one course.

Courses

A course is a collection of scheduled events. It is associated with a Course Type, which defines the number of events in the course and, by association with Credit Blocks, pricing for the course.

Students will initially book for a course and will be booked into all of its events. They can choose to cancel from individual events, but won't be given credit back to their credit block to use for other events not in that course (although they will be able to re-book for the course event).

Payment Plans

Credit Blocks

Credit blocks define pricing and come in two varieties

  • Drop In Credit Blocks which can be used for a number of events of a specific event type with a fixed time period
  • Course Credit Blocks which can be used for a single course of a specifc course type

Subscriptions

Subscriptions are one-off or recurring plans that can (but don't have to be) linked to event types. They can be used as a membership that allows students to purchase classes/events of particular types (with limits, if required), or they can be a simple single payment plan for some other service (e.g. access to resources, online videos etc).

One-off subscription start on a fixed date and run for one period only. Recurring subscriptions will automatically recur for the next subscription period and will be purchaseable by students as long they remain set as "active". Recurring subscriptions do not automatically bill users; 3 days before a subscription is due for renewal, students will be sent a reminder email, and a subscription for the next period will be automatically created and added to their shopping cart for them to purchase if they choose to.

Each Subscription has an optional "current subscriber information" field. This is ONLY visible to current subscribers - useful for subscriptions that involve password access to a resource, for example. You can change this information at the beginning of a new subscription period, and it will only be available to students who are currently subscribed - it will be hidden for any expired subscriptions.

Creating Scheduled Events

There are a number of ways to create scheduled events, once your tracks and event types are set up.

  1. Go to Events > Add Event to create a one-off event
  2. Clone an already created event
  3. Create a timetable of weekly sessions to bulk upload

Cloning Events

This lets you set up an event once, and then duplicate it many times. From the Events page, click the icon to clone an event. You can select from options to clone the event at recurring days/times every week, at recurring intervals on a specific date, or as a single specific date and time.

Timetabled Sessions and Upload

Timetabled Sessions are similar to Events, but have a day of the week and time rather than a specific dates. Once set up, you can choose the dates you want to upload the entire timetable at once. Timetabled sessions can also be cloned, if you're creating multiple similar sessions.

Note that uploaded, clones and newly created events are not immediately visible on the site by default. They can be switched to visible on the Events list page when you're happy with them.

Creating Courses

Courses can be set up before or after their scheduled events are created. The course list page will show you all courses that have been created, and if they have been fully configured (i.e. have the expected number of events assigned to them). Courses are also initially not visible on the site by default, and cannot be set to visible until fully configured.

Details that can be set on a course and an event - such as max participants and visiblity - are overriden by course settings when an event is added to a course. I.e. if you create an event with 10 max participants, and add it to a course that has 8 max participants, the event will be updated with the course setting.

New courses can also be created by cloning a previous course, to copy over course description and other settings. The new course will not be visible on the site by default, and will have no events assigned to it.

Managing Events

Cancelling

If necessary, you can cancel an event from the Events page using the "Cancel" buttons. You will be asked for confirmation first, and students booked into the event will be emailed notification (plus and optional message that you can add during on the cancellation page).

Contacting Students

From the Events and Courses pages, you also have the option to email students booked into an event/course. Click the icon for the event you want to to email.

Registers

Registers give you a list of booked students for each event, with buttons to mark students as attended or not attended. Note that students who cancel after the cancellation period are marked as booked but not attended and will be visible (greyed out) on the registers.
